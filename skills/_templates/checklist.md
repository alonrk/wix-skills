# Skill QA Checklist

- [ ] Skill folder name matches frontmatter `name`.
- [ ] Frontmatter includes valid `name` and `description`.
- [ ] `name` is lowercase-hyphen format and within length limits.
- [ ] Description clearly states what the skill does and when to use it.
- [ ] Includes explicit Wix MCP API-reference-first flow.
- [ ] Includes fallback to docs search/article/schema path.
- [ ] States endpoint/body must come from docs context (no guessing).
- [ ] Includes error handling for missing app and Wix Code not enabled.
- [ ] Includes source docs URLs, with recipe links marked optional.
- [ ] File is concise and domain-specific.
